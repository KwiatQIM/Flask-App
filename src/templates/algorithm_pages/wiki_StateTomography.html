StateTomography

The main function that our library is centered around. After initializing a Tomography object with the number of qubits in the constructor, the StateTomography function can be called.

Inputs:

  • measurements
    • A matrix that stores the states that were used for measurements.
  • counts
    • An array with a length equal to the number of measurements taken. Each value corresponds to the number of counts measured for the measurement basis in the same row in the measurements input.
  • (optional)
    • crosstalk
    • efficiency
    • time
    • singles
    • window
    • error
    • intensities
    • method: ['MLE','HMLE','BME','LINEAR']
      • See StateTomography for more info

Step 1:

The buildTomoInput function is used to make a large tomography input matrix from the measurements, counts, and other optional parameters.

The matrix is then passed into the StateTomography_Matrix Function where the next steps take place.

Step 2:

The large tomography input matrix is given to the filter_data function.

The filter_data function returns four arrays:

  • coincidences
  • measurement state density matrices
  • measurement pure states
  • accidentals

Step 3:

If a starting tomography state is not specified in the Conf Settings (See Conf_File, setConfSetting, or importConf), the linear_tomography function uses linear techniques to find a good starting state for tomography. If the 'LINEAR' method is specified, then this linear tomography is what is returned from StateTomography().

linear_tomography takes the coincidences and measurements as inputs and returns a density matrix corresponding to the starting state and an intensity value that was used to normalize the state

Step 4:

If 'MLE' is the specified method, the maximum_likelihood_tomography function performs maximum likelihood estimation to find the state that corresponds to the maximum probability that the observed measurements occured.


If 'HMLE' is the specified method, the tomography_HMLE function uses hedged maximum likelihood estimation to find desired density matrix.


If 'BME' is the specified method, the tomography_BME function uses bayesian methods to find the desired density matrix.

These functions are all passed the starting matrix, coincidences, measurement density matrices, accidentals, and norms(used for drift correction). They all returns the final density matrix, intensity value used to normalize the state, and the final optimization difference

Return Values:

  • Density Matrix
    • The matrix corresponding to the state that most likely to have produced the observed measurements.
  • Intensity
    • The predicted overall intensity that was used to normalize the state.
  • Final Optimization Value
    • The final value of the optimization function that was used to calculate the most likely state.
