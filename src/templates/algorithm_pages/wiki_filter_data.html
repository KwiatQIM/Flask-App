filter_data

The main purpose of this function is to generate useful data structures from the single tomo_input matrix. The structures are then used to perform the tomography later in the algorithm.

Inputs:

  • tomo_input
    • The large input matrix referenced in StateTomography function. The format for the input can be found here.

Step 1:

The function extracts the time values, singles, and coincidences from the tomo_input matrix.

Since these are just columns in the tomo_input matrix, this process is relatively simple.

Step 2:

If the accidental correction setting in the Conf Settings is set to 1, filter_data uses the window and efficiency settings to calculate the accidental counts.

Step 3:

filter_data uses the crosstalk setting and the measurements to generate the matrices representing the measurement pure states and the measurement density matrices.

Return Values:

  • Coincidences
    • The array of coincidence counts measured.
  • Measurement Density Matrices
    • A three dimensional matrix representing the Density matrices of the states used to take measurements.
  • Measurement Pure States
    • A two dimensional matrix housing the pure state representations of the measurement states.
  • Accidental Counts
    • An array that holds the numbers of projected accidental counts for each measurement state.
